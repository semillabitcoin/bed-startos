# BED — Bitcoin Encrypted Backup (StartOS)

[![Latest release](https://img.shields.io/github/v/release/semillabitcoin/bed-startos?label=release&color=blue)](https://github.com/semillabitcoin/bed-startos/releases/latest)

[![StartOS](https://img.shields.io/badge/StartOS-0.4.0-orange)](https://start9.com)

StartOS `.s9pk` wrapper for [BED — Bitcoin Encrypted Backup](https://github.com/semillabitcoin/descriptor-cifrado).

BED encrypts a multisig descriptor using any cosigner xpub (AES-256-GCM), producing a `.bed` binary
that only the holder of that xpub can decrypt. Compatible with the draft BIP
[bitcoin/bips#1951](https://github.com/bitcoin/bips/pull/1951) and Liana wallet interop.

---

## TL;DR

> **Golden rule (DOC-02):**
> Never store a `.bed` file and a cosigner xpub of the same multisig in the same location.
> If an attacker finds both, they can decrypt the descriptor and recover your full wallet structure.

Install on StartOS → open the Tor or LAN URL from the dashboard → paste descriptor → encrypt → distribute
one `.bed` copy per cosigner location.

## Install

### Sideload (always available)

Download the `.s9pk` for your device architecture from the
[latest GitHub Release](https://github.com/semillabitcoin/bed-startos/releases/latest):

- `bed_x86_64.s9pk` — for StartOS on x86\_64 hardware (Intel/AMD)
- `bed_aarch64.s9pk` — for StartOS on ARM hardware (Raspberry Pi, Purism, etc.)

Then sideload via StartOS UI ("System → Sideload Package") or CLI:

```bash
start-cli package install -s bed_<arch>.s9pk
```

### Verifying the build

Each release pins the upstream `descriptor-cifrado` image by digest. To
confirm a downloaded `.s9pk` matches the digest published for that release:

```bash
start-cli s9pk inspect bed_<arch>.s9pk manifest | grep dockerTag
```

The digest must match the entry for that version in [CHANGELOG.md](CHANGELOG.md).

### Registry (Semilla Bitcoin)

When published to the Semilla Bitcoin registry, install by adding the registry URL in StartOS
Settings → Registry, then searching for "BED".

## Usage

1. Open the **Tor** onion or **LAN** `.local` URL shown on the StartOS dashboard.
2. In the **Cifrar** tab, paste a multisig descriptor (must use `<0;1>/*` derivation).
3. Paste a cosigner xpub as the encryption key.
4. Click **Cifrar** — download the `.bed` binary, copy the armored text, or scan the QR.
5. Store the `.bed` in a location separate from the xpub used to encrypt it.

For full usage documentation including decrypt instructions and the history feature:
[Full usage docs](https://github.com/semillabitcoin/descriptor-cifrado#usage)

## Threat Model

BED protects descriptor privacy during distribution. A `.bed` file encrypted with xpub A reveals
nothing to anyone who does not hold xpub A — the AES-256-GCM ciphertext is indistinguishable from
random bytes without the key derived from that xpub.

> **Golden rule (repeated):**
> Never store a `.bed` file and a cosigner xpub of the same multisig in the same location.
> Each physical location should hold either the `.bed` OR the relevant xpub, never both.
> The security model requires that co-location (`.bed` + `xpub_N`) never occurs at the same site.

What BED does NOT protect against:
- A compromise of the StartOS device during an active encrypt session (descriptor passes through memory in cleartext).
- An attacker who already holds the matching cosigner xpub for a given `.bed`.
- Loss of all cosigner xpubs simultaneously (the `.bed` becomes permanently undecryptable).

[Full threat model](https://github.com/semillabitcoin/descriptor-cifrado#threat-model)

## What's in the s9pk

- **Image:** `ghcr.io/semillabitcoin/descriptor-cifrado` (digest-pinned at build time — deterministic, no floating tags)
- **Architectures:** `x86_64` and `aarch64` (separate `.s9pk` files)
- **Volume:** `main` mounts at `/data/encrypted/` inside the container; preserved across updates and
  included in StartOS backups automatically
- **Health check:** `checkPortListening` on port 8080 — marks the service healthy when the axum server is ready
- **Interfaces:** Tor onion + LAN `.local` auto-generated by StartOS from a single `bindPort(8080, 'http')` declaration
- **Dependencies:** None — BED has no cross-package StartOS dependencies

## Building from Source

Requirements: `node` 20+, `pnpm` 9+ (`npm install -g pnpm`), `start-cli` 0.4.0-beta.5+, `~/.startos/developer.key.pem` (from `start-cli init-key`).

```bash
git clone https://github.com/semillabitcoin/bed-startos.git
cd bed-startos
pnpm install --frozen-lockfile
make clean x86 arm   # produces bed_x86_64.s9pk and bed_aarch64.s9pk
```

Note: `make` pulls the GHCR image at pack time. If the image is private, authenticate first:
```bash
echo "$GHCR_PAT" | docker login ghcr.io -u <username> --password-stdin
```
